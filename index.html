Technical Insight: Eliminating Hallucination Risk

Large Language Models are excellent at semantic extraction, but they cannot do math reliably. In this build, we completely locked Claude out of arithmetic. Claude extracts the raw figures from scanned balance sheets into strict JSON, then passes those variables to our deterministic Python Model Context Protocol (MCP) server. The Python server calculates DSCR and LTV, returning mathematically perfect results. The FCA gets an auditable process; the bank gets zero hallucination risk.

Logging records that something happened. ComplyAI verifies what happened. Source Tracing checks every figure against source documents and rejects fabricated citations, not just logs them. The Immutable Audit Log is hash-chained, so tampering with a historical entry is cryptographically detectable, not just discouraged by access controls. Standard logging cannot make either claim.

For every decision, ComplyAI records the source documents referenced, the verification outcome for every figure, the plain-language explanation generated, and the named individual who reviewed and signed off before the decision was finalised. This is hash-chained so no historical entry can be silently altered.

The SMCR Accountability Gate requires a specific person to review and sign off before a consequential decision is finalised, and once signed off, that decision cannot be silently overwritten by a second reviewer. This is designed specifically to answer the SMCR accountability question regulators are asking UK banks directly.

Security and architecture

No. The system runs on your own Anthropic API account and your own infrastructure, not ours, so ShiftAi never holds your data at all. Under Anthropic's standard commercial terms, API inputs and outputs are not used to train its models. Where your organisation requires a Zero Data Retention agreement with Anthropic directly, we can build to that requirement and confirm it as part of the AI Readiness Assessment.
We use a strict DevSecAI architecture. The AI operates in a sandboxed environment where read and write permissions are physically separated. Any action that alters a database, sends an email to a client, or approves a credit line requires a deterministic Human-in-the-Loop (HITL) interrupt. The AI can draft the decision, but a human must click approve.
We do not allow the LLM to do mathematics. We use the LLM purely for semantic extraction and orchestration. It extracts the raw figures from the document and passes them to a deterministic, tested Python Model Context Protocol (MCP) server to calculate DSCR, LTV, or affordability. The arithmetic is deterministic and independently testable, and the LLM simply formats the result for the report.
Because we build using abstracted frameworks like the Model Context Protocol (MCP), your system architecture is completely decoupled from the underlying LLM. If a model is deprecated, we update the API pointer, run your automated evaluation suite (LLM-as-a-judge) to ensure output quality remains identical, and push the update seamlessly.
